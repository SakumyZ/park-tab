# Park Tab 需求文档

## 1. 产品定位

Park Tab 是一个面向多设备使用者的浏览器标签页管理插件。核心目标是复刻 OneTab 的高频能力：一键收纳当前标签页、按分组保存、恢复标签页、节省浏览器资源；同时补足 OneTab 在多设备同步、完整状态迁移、结构化导入导出上的不足。

产品应优先保证以下体验：

- 保存标签页后，用户可以清楚看到标签属于哪个分组、保存时间、标题、URL 和状态。
- 同一账号或同一 WebDAV 存储下，多台设备看到的保存状态应尽量一致。
- 导出数据再导入新设备后，标签、分组、排序、锁定、归档等状态必须完整保留。
- 本地优先可用：离线时仍可保存和管理标签页，联网后再同步。
- 数据格式开放、可读、可备份，避免用户被插件锁定。

## 2. 目标用户

- 经常同时打开大量标签页，需要定期收纳和恢复的人。
- 在多台电脑之间切换工作环境，希望标签保存状态自动同步的人。
- 不信任封闭云服务，倾向使用自有 WebDAV、NAS、坚果云、Nextcloud 等同步方案的人。
- 需要把标签页按项目、任务、上下文长期归档的人。

## 3. 竞品痛点与改进方向

### OneTab 主要能力

- 一键将当前窗口或全部窗口标签转换为列表。
- 从列表中恢复单个标签或整组标签。
- 支持命名、锁定、分享、导入导出等基础能力。

### 需要弥补的问题

- 云同步能力弱，多设备一致性依赖手动导入导出。
- 导出格式不完整，容易丢失分组、排序、锁定等状态。
- 冲突处理不透明，多设备编辑时难以判断哪份数据可信。
- 长期使用后检索、分类、批量管理能力不足。
- 备份和迁移缺少稳定、版本化的数据格式。

## 4. 范围定义

### MVP 必须完成

- 浏览器扩展基础框架，优先支持 Chromium 系浏览器，兼容 Manifest V3。
- 一键保存当前标签页到新分组。
- 保存当前窗口、全部窗口、选中标签页。
- 分组列表页面，可查看、重命名、排序、锁定、删除分组。
- 标签项列表，可查看标题、URL、favicon、保存时间，可单个恢复、删除、移动。
- 恢复整组标签页，支持恢复后保留或删除已保存记录。
- 本地存储，支持离线使用。
- WebDAV 同步配置、连接测试、手动同步。
- 自动同步：启动时、保存后、恢复/修改后按节流策略同步。
- 完整导出和导入，导入后设备间显示状态一致。
- 数据格式带 schemaVersion，支持后续迁移。

### V1 增强

- 多设备冲突检测与合并预览。
- 分组搜索、标签搜索、按域名筛选。
- 标签页去重，支持同组去重和全局去重。
- 分组置顶、归档、标签数量统计。
- 自动保存规则，例如按窗口、域名、时间生成分组名。
- 定期自动备份，保留最近 N 个快照。
- WebDAV 同步加密，可选用户自定义密码。

## 5. 核心概念

### Workspace

一个用户的数据空间。MVP 可以只有默认工作区，后续支持多个工作区。

### Group

一次保存动作产生的标签组，也可以由用户手动创建。分组是导入导出和同步一致性的核心单位。

### Tab Item

被保存的单个标签页。必须保存标题、URL、favicon、创建时间、更新时间、排序、状态等信息。

### Device

产生或修改数据的浏览器实例。每个设备生成稳定 deviceId，用于同步冲突判断和审计。

### Snapshot

一次完整数据备份。用于导出、自动备份、冲突恢复。

## 6. 功能需求

### 6.1 标签保存

- 用户可以通过扩展图标或右键菜单保存标签页。
- 支持保存范围：
  - 当前标签页。
  - 当前窗口全部标签页。
  - 所有窗口全部标签页。
  - 当前选中的多个标签页。
- 保存后默认关闭已保存标签页，可在设置中关闭此行为。
- 保存时应自动生成分组名称，例如 `2026-06-01 设计资料` 或 `未命名分组`。
- 保存时应记录标签原始窗口、原始索引、是否 pinned、是否 active。
- 特殊页面处理：
  - `chrome://`、扩展页面、空白页等不可恢复页面应提示或跳过。
  - 重复 URL 可按用户设置保留或去重。

### 6.2 分组管理

- 支持重命名分组。
- 支持锁定分组。锁定后不能被批量删除、恢复后自动删除、自动去重误删。
- 支持删除分组，删除前需要确认；锁定分组需要二次确认。
- 支持展开/折叠分组，并同步折叠状态。
- 支持拖拽调整分组排序，并同步排序。
- 支持归档分组，归档后默认隐藏。
- 支持给分组添加备注。

### 6.3 标签项管理

- 支持单个标签恢复到当前窗口或新窗口。
- 支持整组恢复到当前窗口或新窗口。
- 支持恢复后是否删除保存记录：
  - 默认保留锁定分组。
  - 非锁定分组按设置决定是否在整组恢复后自动删除记录。
  - 非锁定分组按设置决定是否在单标签恢复后自动删除该条记录；分组被清空时自动删除空分组。
- 支持删除单个标签。
- 支持移动标签到其他分组。
- 支持拖拽调整标签排序。
- 支持编辑标签标题。
- URL 不建议直接编辑；如支持，必须有校验和历史记录。

### 6.4 搜索与筛选

- 支持按标题、URL、分组名搜索。
- 支持按域名筛选。
- 支持只看锁定、归档、最近保存、最近恢复。
- 搜索结果应保留分组上下文。

### 6.5 导出

导出必须保留完整状态，不能只导出 URL 列表。

- 支持导出完整 JSON 文件。
- JSON 必须包含：
  - schemaVersion。
  - workspace 信息。
  - groups。
  - tabs。
  - settings 中与显示状态相关的必要字段。
  - device 元数据。
  - exportedAt。
- 导出数据应可直接作为同步数据快照使用。
- 可选支持简洁 HTML/Markdown 导出，但这类格式只用于阅读，不作为完整迁移格式。

### 6.6 导入

- 支持导入 Park Tab 完整 JSON。
- 导入后应完整恢复：
  - 分组名称。
  - 分组排序。
  - 分组锁定、归档、折叠、备注。
  - 标签标题、URL、favicon、排序。
  - 标签保存时间、更新时间、恢复时间。
  - 设置中的显示相关状态。
- 导入策略：
  - 替换当前全部数据。
  - 合并到当前数据。
  - 作为新工作区导入，V1 可实现。
- 导入前必须校验 schemaVersion 和数据完整性。
- 导入前应自动创建本地备份，避免误覆盖。

### 6.7 WebDAV 同步

- 支持配置 WebDAV 地址、用户名、密码或应用密码。
- 支持连接测试。
- 支持设置远程路径，例如 `/park-tab/sync.json`。
- 支持手动同步。
- 支持自动同步：
  - 扩展启动后同步。
  - 本地数据变更后延迟同步。
  - 定时同步，例如每 5-30 分钟。
- 同步状态展示：
  - 从未同步。
  - 同步中。
  - 同步成功及时间。
  - 同步失败及原因。
  - 存在冲突。
- 同步必须本地优先。网络失败不应阻止本地保存和恢复。
- WebDAV 远程文件建议包含：
  - `sync.json`：当前数据。
  - `backups/YYYYMMDD-HHmmss.json`：快照备份，可选。
  - `lock.json`：同步锁，可选，避免多设备同时写入。

### 6.8 冲突处理

MVP 可先实现保守策略：

- 每条记录包含 `id`、`createdAt`、`updatedAt`、`deletedAt`、`revision`、`updatedByDeviceId`。
- 同步前拉取远程数据。
- 如果远程 revision 等于本地 baseRevision，直接上传本地新版本。
- 如果本地和远程都有变更，进入合并流程。
- 合并原则：
  - 不同分组或不同标签的变更自动合并。
  - 同一字段冲突优先保留更新时间更晚的一方，并记录冲突日志。
  - 删除与编辑冲突时，锁定分组/标签优先保留；非锁定项可进入冲突列表。
- 冲突发生时不得静默丢数据，至少保留本地和远程快照。

V1 应提供冲突预览界面，让用户选择本地、远程或合并结果。

### 7.2 ID 与时间

- ID 使用 UUID 或 Nano ID，不能依赖浏览器 tabId/windowId。
- 所有时间使用 ISO 8601 UTC 字符串。
- 排序使用数值 `sortOrder`，预留间隔，方便拖拽插入。

### 7.3 软删除

- 删除分组或标签时优先设置 `deletedAt`。
- 同步合并后再按保留策略清理软删除数据。
- 软删除可以避免多设备同步时被删除数据重新出现或误删编辑。

## 8. 非功能需求

### 8.1 可靠性

- 保存标签页是最高优先级操作，不能因同步失败而失败。
- 导入、同步、批量删除前应创建本地快照。
- 所有数据迁移必须可回滚。

### 8.2 安全

- WebDAV 密码使用浏览器安全存储能力保存，避免明文出现在导出文件中。
- 完整导出默认不包含 WebDAV 密码。
- V1 支持端到端加密同步文件。
- 日志不能打印密码、token、完整认证头。

### 8.3 性能

- 100 个分组、10000 个标签下列表仍应可搜索和滚动。
- 大列表应考虑虚拟滚动。
- 同步和导入不能阻塞主要 UI。

### 8.4 兼容性

- 首版支持 Chrome、Edge、Brave 等 Chromium 浏览器。
- Firefox 可作为后续目标，需要评估 Manifest V3 API 差异。

## 9. 页面与交互

### 9.1 Popup

- 快速保存当前标签、当前窗口、全部窗口。
- 显示最近一次同步状态。
- 提供打开管理页入口。

### 9.2 管理页

- 左侧或顶部提供搜索、筛选、同步状态、设置入口。
- 主区域展示分组和标签。
- 分组操作包括重命名、锁定、归档、删除、恢复整组。
- 标签操作包括打开、删除、移动、复制 URL。

### 9.3 设置页

- 常规设置：
  - 保存后关闭标签页。
  - 恢复后删除记录。
  - 默认恢复到当前窗口或新窗口。
- 同步设置：
  - WebDAV URL。
  - 用户名。
  - 密码或应用密码。
  - 远程路径。
  - 自动同步开关和间隔。
  - 连接测试。
- 数据管理：
  - 导出完整 JSON。
  - 导入 JSON。
  - 创建本地备份。
  - 清理软删除数据。

## 10. 权限需求

浏览器扩展预计需要以下权限：

- `tabs`：读取和创建标签页。
- `tabGroups`：如需兼容浏览器原生标签组，可选。
- `storage`：保存本地数据和设置。
- `contextMenus`：右键菜单保存标签页。
- `alarms`：定时同步。
- `notifications`：同步失败或冲突提醒，可选。
- WebDAV 远程域名 host permissions：访问用户配置的 WebDAV 服务。

权限申请应尽量按需，并在设置页解释用途。

## 11. 同步流程草案

### 11.1 手动同步

1. 读取本地数据和本地 baseRevision。
2. 从 WebDAV 下载远程 `sync.json`。
3. 校验远程 schemaVersion。
4. 比较本地与远程 revision。
5. 无冲突时合并并上传新 revision。
6. 有冲突时生成冲突报告和本地快照。
7. 更新本地 lastSyncedAt 和同步状态。

### 11.2 自动同步

1. 本地数据变更后标记 dirty。
2. 延迟一段时间触发同步，避免频繁上传。
3. 如果正在同步，合并为下一次同步请求。
4. 同步失败时指数退避重试。
5. 多次失败后只显示状态，不打断保存流程。

## 12. 验收标准

### 12.1 保存与恢复

- 保存当前窗口后，原窗口标签按设置关闭。
- 管理页能看到一个新分组，分组内标签数量、标题、URL、顺序正确。
- 恢复整组后，标签打开顺序与保存顺序一致。
- 锁定分组恢复后不会被自动删除。

### 12.2 完整导入导出

- 在设备 A 创建多个分组，设置锁定、归档、折叠、备注和自定义排序。
- 导出 JSON。
- 在设备 B 导入 JSON。
- 设备 B 显示的分组、标签、排序、锁定、归档、折叠、备注与设备 A 一致。
- 再从设备 B 导出，核心数据与设备 A 导出的数据语义一致。

### 12.3 WebDAV 同步

- 设备 A 保存标签并同步成功。
- 设备 B 手动同步后看到设备 A 的完整数据状态。
- 设备 B 修改分组名和锁定状态并同步。
- 设备 A 再同步后看到同样修改。
- 网络断开时仍能本地保存，恢复网络后可以同步。

### 12.4 冲突保护

- 设备 A 和 B 离线分别修改同一分组。
- 两台设备恢复网络同步后，不应静默丢失任一侧数据。
- 应至少保留冲突前本地快照和远程快照。

## 14. 已决策问题

- 首版只支持 Chromium 系浏览器，后期再兼容 Firefox。
- 浏览器原生 Tab Groups 的保存与恢复后续支持，首版不做。
- WebDAV 同步文件首版不加密，后续再支持可选加密。
- 冲突合并首版先做自动合并，不提供用户可视化选择界面。
- 不支持多个工作区，首版和后续默认都以单工作区为主。
- UI 技术栈使用 Svelte + Tailwind CSS。
- 由于项目是浏览器插件，开发时需要优先控制包体积和运行性能。
